Git Command List

Install git

ก่อนที่จะใช้งานคุณต้องติดตั้ง Git ก่อน โดยคุณสามารถทำได้หลายวิธี วิธีหลักสองวิธีคือติดตั้งจากซอร์สโค้ดหรือติดตั้งจากแพคเกจที่มีอยู่แล้วสำหรับระบบปฏิบัติการของคุณ สำหรับ Windows สามารถ download และติดตั้ง Git ได้จาก GitHub page

Syntax

            download  http://msysgit.github.com/

git init

คำสั่งนี้จะสร้างแฟ้มข้อมูลย่อยชื่อ .git ซึ่งเก็บแฟ้มโครงสร้างของ Git repository เปล่า ๆ ที่ยังไม่มีเนื้อหาใด ในขั้นตอนนี้แฟ้มที่อยู่ในโปรเจคของคุณจะยังไม่ถูก track โดย Git

Syntax

            $git init [-q | --quiet] [--bare] [--template=<template_directory>]
	  [--separate-git-dir <git dir>]
	  [--shared[=<permissions>]] [directory]

git add

คำสั่งเพื่อเพิ่ม change เข้าไปเตรียมพร้อมสำหรับการ commit

Syntax

            git add
            [--verbose | -v] [--dry-run | -n] [--force | -f] [--interactive | -i] [--patch | -p]
            [--edit | -e] [--[no-]all | --[no-]ignore-removal | [--update | -u]]
            [--intent-to-add | -N] [--refresh] [--ignore-errors] [--ignore-missing]
            [--chmod=(+|-)x] [--]

git commit

คือการสร้าง snapshot ให้กับ repository ทำให้เราสามารถย้อนกลับมาดูว่าเราเปลี่ยนหรือแก้ไขโค๊ดอะไรไปบ้างได้

Syntax

git commit [-a | --interactive | --patch] [-s] [-v] [-u<mode>] [--amend]
	   [--dry-run] [(-c | -C | --fixup | --squash) <commit>]
	   [-F <file> | -m <msg>] [--reset-author] [--allow-empty]
	   [--allow-empty-message] [--no-verify] [-e] [--author=<author>]
	   [--date=<date>] [--cleanup=<mode>] [--[no-]status]
	   [-i | -o] [-S[<keyid>]] [--] [<file>…​]

git push

หลังจากเราทำงานไปได้พักนีง เราควรจะส่ง history ใหม่ๆจาก clone repo ของเรากลับไปยัง origin push จะลอก history ของเรากลับไปยัง origin repo

Syntax

            git push [NICKNAME] [BRANCH]

git pull

ใช้ดึงความเปลี่ยนแปลงจาก remote มายัง local และรวมเข้าด้วยกัน (มีค่าเท่ากับ fetch+merge)

Syntax

            $git pull [options] [<repository> [<refspec>…​]]

git checkout

เปลี่ยน Branch หรือ ยกเลิก change ที่ยังไม่ได้ commit

Syntax

            git checkout [-q] [-f] [-m] --detach [<branch>]
            git checkout [-q] [-f] [-m] [--detach] <commit>
            git checkout [-q] [-f] [-m] [[-b|-B|--orphan] <new_branch>] [<start_point>]
            git checkout [-f|--ours|--theirs|-m|--conflict=<style>] [<tree-ish>] [--] <paths>…
            git checkout [-p|--patch] [<tree-ish>] [--] [<paths>…​]

git clone

ในกรณีที่เราไปเจอโปรเจ็ค cool cool น่าสนใจ เราสามารถที่จะเซฟมาลงเครื่องของเราได้เลย เราเรียกการทำแบบนี้ว่าการ clone เหมือนกับการกดดาวน์โหลดโปรเจ็คมานั่นแหละครับ

Syntax

git clone [--template=<template_directory>]
	  [-l] [-s] [--no-hardlinks] [-q] [-n] [--bare] [--mirror]
	  [-o <name>] [-b <name>] [-u <upload-pack>] [--reference <repository>]
	  [--dissociate] [--separate-git-dir <git dir>]
	  [--depth <depth>] [--[no-]single-branch]
	  [--recursive | --recurse-submodules] [--[no-]shallow-submodules]
	  [--jobs <n>] [--] <repository> [<directory>]

Syntax

            $git status [<options>…​] [--] [<pathspec>…​]

git diff

แสดงส่วนของโค้ดที่แตกต่างระหว่างแต่ละ commit และอื่นๆ

Syntax

            git diff [options] [<commit>] [--] [<path>…​]
            git diff [options] --cached [<commit>] [--] [<path>…​]
            git diff [options] <commit> <commit> [--] [<path>…​]
            git diff [options] <blob> <blob>
            git diff [options] [--no-index] [--] <path> <path>
